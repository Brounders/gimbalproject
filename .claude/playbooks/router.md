# Claude Router

Используй этот файл как карту выбора playbook по естественной формулировке Human.

## Маршрутизация

- Если запрос содержит смысл:
  - `план`, `аудит`, `создай задачи`, `обнови active plan`, `прими отчет`, `проверь Claude`
  - Тогда сначала открыть `orchestrator.md`.

- Если запрос содержит смысл:
  - `статус RTX`, `что с обучением`, `resume`, `эпохи`, `почему остановилось`, `last.pt`, `best.pt`
  - Тогда сначала открыть `rtx_intake.md`.

- Если запрос содержит смысл:
  - `quality gate`, `benchmark`, `baseline`, `candidate`, `сравни метрики`, `можно ли принимать модель`
  - Тогда сначала открыть `quality_gate.md`.

- Если запрос содержит смысл:
  - `запусти обучение`, `подготовь промт для RTX`, `термобезопасность`, `продолжи с последней точки`, `curriculum`
  - Тогда сначала открыть `training_ops.md`.

- Если запрос содержит смысл:
  - `интерфейс`, `окно`, `PySide6`, `панели`, `удобство оператора`, `HUD`, `video stage`
  - Тогда сначала открыть `pyside6_ui.md`.

## Правило по умолчанию

Если запрос широкий и смешанный:
1. Сначала открыть `orchestrator.md`.
2. Затем открыть один дополнительный playbook по доминирующему контуру задачи.
3. Не загружать все playbooks сразу без необходимости.
