# Training Ops Playbook

## Когда использовать
- Нужно подготовить или скорректировать обучение на RTX.
- Нужно продолжить run с последней точки.
- Нужно снизить термориск.

## Базовые правила
- Один train job одновременно.
- Не плодить новые run-директории без необходимости.
- По умолчанию продолжать существующий run от `last.pt`.
- Всегда требовать epoch accounting в статусе.

## Термобезопасность
- Снижать CPU pressure прежде, чем трогать batch/model logic.
- Предпочтительно:
  - `workers=0`
  - ограничение CPU threads
  - `cache none`
  - умеренный batch с fallback
- При критическом перегреве стоп должен быть контролируемым, с сохранением checkpoint и лога.

## Мак-контур после RTX
- Принять артефакты.
- Проверить baseline/candidate.
- Только после gate решать, использовать ли модель дальше.
