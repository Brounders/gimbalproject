# GimbalProject — Автоматическая система обнаружения и сопровождения БПЛА

## Обзор проекта

Десктоп-приложение (Python + PySide6) для автоматического обнаружения и трекинга БПЛА в режимах день/ночь/ИК. Целевая платформа деплоя: Raspberry Pi 5 + AI HAT+ (Hailo).

## Архитектура

```
┌─────────────────────────────────────────────────┐
│               Оператор-UI (PySide6)             │
│  Русскоязычный интерфейс, минимум контролов     │
├─────────────────────────────────────────────────┤
│            Модуль управления (Controller)        │
│  Координация детекции, трекинга, UI              │
├──────────────────┬──────────────────────────────┤
│  Детекция (YOLO) │  Трекинг (Kalman + логика)   │
│  День/Ночь/ИК    │  Захват, удержание, потеря   │
├──────────────────┴──────────────────────────────┤
│         Видеопоток (камера / файл / RTSP)        │
├─────────────────────────────────────────────────┤
│     Деплой: RPi5 + Hailo AI HAT+ / x86 dev      │
└─────────────────────────────────────────────────┘
```

## Стек технологий

- **Язык:** Python 3.11+
- **UI:** PySide6 (десктоп, не веб)
- **Детекция:** YOLO (ultralytics), поддержка ONNX/Hailo экспорта
- **Трекинг:** Фильтр Калмана, кастомная логика удержания
- **Видео:** OpenCV
- **Целевое железо:** Raspberry Pi 5 + AI HAT+ (Hailo-8L)
- **Среда разработки:** macOS (основная), Windows (обучение моделей, RTX 3070)

## ⚠️ КРИТИЧЕСКИЕ ПРАВИЛА БЕЗОПАСНОСТИ ⚠️

### Защита основной ветки

1. **ЗАПРЕЩЕНО** коммитить напрямую в `main` (или текущую рабочую ветку)
2. **ВСЯ** работа agent team ведётся ТОЛЬКО на ветке `team/<название-задачи>`
3. **Перед началом** любой работы — создать ветку от текущего HEAD основной ветки
4. **Мерж в main** выполняет ТОЛЬКО Bround вручную, НИКОГДА агенты
5. **Если сомневаешься** — спроси Bround, не действуй

### Защита файлов (НЕ ТРОГАТЬ)

- `CLAUDE.md` (этот файл)
- `agents/*.md` (описания ролей)
- `.claude/` (настройки Claude Code)
- Файлы конфигурации в корне проекта

### Процедура работы агентов

```
1. git stash (если есть незакоммиченные изменения)
2. git checkout -b team/<задача> main
3. [работа агентов — только в этой ветке]
4. git add . && git commit -m "[модуль] описание"
5. Сообщить Bround: "Готово, ветка team/<задача>"
6. Bround ревьюит → мержит ИЛИ удаляет ветку
7. git checkout main (вернуться)
```

## Ключевые принципы

1. **Стабильность трекинга** — приоритет №1
2. **Простота UI** — интуитивный, русский, минимум контролов
3. **Деплой на edge** — RPi5 + Hailo, производительность критична
4. **Автоматизация** — предпочитать автоматизированные решения
5. **Безопасность прогресса** — работа на ветках, main защищён

## Правила для всех агентов

- **Язык кода:** Python, английские имена переменных/функций/классов
- **Язык комментариев:** русский
- **Язык UI:** русский
- **Коммиты:** на русском, формат `[модуль] описание`
- **Ветки:** только `team/<название>`, НИКОГДА `main`
- **Нет ломающих изменений** без согласования с менеджером
- **Тесты обязательны** для критической логики
- **Type hints** везде
- **Docstrings** для публичных функций (русский)
