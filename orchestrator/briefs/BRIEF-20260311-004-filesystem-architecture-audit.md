# BRIEF: BRIEF-20260311-004-filesystem-architecture-audit

## Context
Направление Human: провести аудит файловой системы проекта. Требуется понять, какие директории/файлы относятся к ядру, какие к операционным артефактам, где есть технический долг структуры, и какие безопасные шаги реорганизации возможны без риска для runtime.

## Objective
Сформировать инженерно-проверяемый аудит ФС и план безопасной реорганизации:
1) инвентаризация структуры и размеров,
2) классификация файлов по роли и критичности,
3) подготовка безопасного плана cleanup/архивации без удаления рабочих данных.

## Success Metrics
- Есть воспроизводимый отчет по структуре и весу директорий.
- Есть классификация "ядро / runtime / training-artifacts / legacy / candidate cleanup".
- Есть пошаговый безопасный план изменений с явными запретами на рискованные действия.

## Boundaries
- Must do:
  - без изменений runtime-кода;
  - без удаления данных в этой фазе;
  - все выводы подтверждены командами.
- Must not do:
  - перемещение/удаление `datasets/`, `runs/`, `tracker_env/`;
  - большие рефакторы исходников;
  - изменения алгоритмов трекинга.

## Deliverables
- 3 локальные задачи Claude для аудита и безопасной реорганизации.
- Обновленные state-файлы orchestration.
