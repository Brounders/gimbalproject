# BRIEF: BRIEF-20260311-001-stability-loop

## Context
Проект находится в фазе `stabilization-first`: качество трекинга ограничено нестабильностью ID на night/IR сценах, при этом требуется сохранить текущую рабочую архитектуру без крупных рискованных изменений.

## Objective
Запустить управляемый цикл из небольших проверяемых шагов:
1) локальная декомпозиция монолита pipeline без смены поведения,
2) добавление базового тестового контура для lock-политики,
3) параллельный training/eval цикл на RTX через curriculum.

## Success Metrics
- Pipeline refactor stage-1 выполнен без изменения runtime-поведения.
- Появились воспроизводимые unit-тесты для lock/ID switch policy.
- RTX запускает очередной curriculum-run и возвращает артефакты для intake.

## Boundaries
- Must do:
  - Минимальные изменения и обратимость.
  - Строгая проверяемость (команды валидации обязательны).
  - Разделение UI/business/runtime/io слоев.
- Must not do:
  - Крупные архитектурные переписывания.
  - Изменение модели работы quality-gate.
  - Внедрение новых зависимостей без необходимости.

## Deliverables
- 2 инженерные задачи Claude с приемочными критериями.
- 1 training-задача RTX с ожидаемыми артефактами и валидацией.
- Обновленные state-индексы orchestrator.
