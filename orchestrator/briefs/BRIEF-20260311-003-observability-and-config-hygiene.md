# BRIEF: BRIEF-20260311-003-observability-and-config-hygiene

## Context
Фаза стабилизации продолжается: базовые задачи по декомпозиции и unit-тестам приняты, но контур эксплуатации все еще уязвим к человеческим ошибкам в профилях и к позднему обнаружению деградаций lock-поведения.

## Objective
Добавить low-risk инженерные инструменты, которые повышают предсказуемость цикла без вмешательства в core-runtime алгоритмы:
1) анализатор lock-событий для сравнения сессий,
2) валидатор YAML-профилей до запуска.

## Success Metrics
- Есть CLI-инструмент для анализа lock-event логов с машинно-читаемым итогом.
- Есть CLI-валидатор профилей, ловящий ошибки ключей/типов до запуска.
- Изменения локальные, проверяемые, без влияния на runtime-логику трекинга.

## Boundaries
- Must do:
  - минимальный diff;
  - фокус на tooling/quality guardrails;
  - валидация через compile/tests/smoke.
- Must not do:
  - изменение decision-loop трекера;
  - изменения quality-gate порогов;
  - крупные рефакторы pipeline.

## Deliverables
- 2 новые задачи для Claude (tooling).
- Обновленные state-файлы orchestration.
