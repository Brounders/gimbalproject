### Target Info Card

**app/ui/video_stage.py** — расширен класс `VideoStage`:
- Добавлен `self._overlays_top_right: list[QWidget]`
- Метод `add_overlay_top_right(widget)` — устанавливает parent, добавляет в список, вызывает позиционирование

- Labels: objectName='TargetCardRow', статус: objectName='TargetCardState'

**app/main_gui.py** — `build_video_stage()`:
```python

self.video_stage.add_overlay_top_right(self.target_info_card)
```

**app/main_gui.py** — `_update_stats()`:
- Отслеживает `self._target_lock_start: float | None` — момент начала TRACK
- Вычисляет elapsed time на цели в формате MM:SS
- Обновляет `_tc_id`, `_tc_conf`, `_tc_fps`, `_tc_time`, `_tc_state` каждый кадр
- Корректно обрабатывает отсутствие цели (IDLE показывает «—»)

## Риски

- `switch_target()` вызывается в worker-thread через флаг (GIL-safe), но между
  установкой флага и чтением проходит до одного frame. Это приемлемо для UI-triggered action.
- При отсутствии видео (worker=None) кнопка disabled — краш невозможен.
- `adjustSize()` в `_reposition_overlays` требует, чтобы card имела non-zero sizeHint при первом
  показе. `setMinimumWidth(170)` обеспечивает это.
