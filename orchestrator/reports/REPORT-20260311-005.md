## Описание утилиты
`analyze_lock_events.py` принимает один или несколько путей к `.jsonl` lock-event логам и выводит табличный summary:

| Метрика | Поле вывода |
|---|---|
| Кол-во событий по типам | `acquired`, `lost`, `reacquired`, `switch` |
| Среднее/макс switches/мин | `switches_per_min_mean`, `switches_per_min_max` |
| Диапазон кадров | `frame_span` |

Флаги CLI: `--output-json PATH`, `--output-csv PATH`.

Функции доступны для импорта в тестах и скриптах без argv: `load_events()`, `summarize()`, `analyze_files()`.

## Residual Risks
- Risk: `frame_span = max - min + 1` предполагает целочисленные `frame_index`. Если источник лога пишет float — `int()` применяется явно, поведение корректно.
- Risk: `lock_switches_per_min` отсутствует в некоторых event-типах (например, `lost` может не нести это поле). `ev.get()` возвращает `None` → поле пропускается, mean/max считается по фактическим значениям.

## Варианты эволюции
1. **Временная шкала** — добавить поле `timestamp` в JSONL и считать реальный elapsed time вместо frame_span; строить time-series график switches/min.
2. **Аномалия-детектор** — добавить флаг `--threshold-switches FLOAT`: выделять сессии, где `switches_per_min_max > threshold`, для автоматического тriage регрессий.
3. **Сравнение baseline/candidate** — режим `--diff BASE CANDIDATE`: diff по каждой метрике с % изменением (аналог `run_quality_gate.py`, но для lock-event дат).
4. **HTML отчёт** — `--output-html PATH` с простой таблицей и sparkline-графиком через stdlib `html` (без внешних зависимостей).
