## Что сделано

Устранена причина TASK-022 Review incomplete: `inspector_module` больше не восстанавливается
из сохранённых UI-настроек и гарантированно остаётся скрытым в operator layer.

### Изменения (app/main_gui.py)

**`_save_app_settings()`** — удалена строка:
```python
# было:
self.settings.setValue('ui/inspector_visible', self.inspector_module.isVisible())
```
Значение больше не персистируется — нет возможности накопить `true` для следующего запуска.

**`_load_app_settings()`** — заменено восстановление из настроек на явное принудительное скрытие:
```python
# было:
inspector_visible = str(self.settings.value('ui/inspector_visible', 'false')).lower() == 'true'
self.inspector_module.setVisible(inspector_visible)

# стало:
self.inspector_module.setVisible(False)  # always hidden in operator layer
```

Если в QSettings осталось старое значение `ui/inspector_visible=true` от прошлых запусков,
оно теперь игнорируется (ключ не читается).

### Expert-контур

`_toggle_expert_mode()` и Expert dialog не тронуты. Диагностический drawer по-прежнему
доступен через Expert dialog (если пользователь сознательно откроет его).
