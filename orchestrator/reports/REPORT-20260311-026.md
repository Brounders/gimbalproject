## Что сделано

Расширен `_adapt_auto_scene` с Day/Night до полного Day/Night/IR маршрутизатора
через saturation-based heuristic.

### Алгоритм (TASK-026)

**config.py** — новые поля:
| Поле | Default | Описание |
|------|---------|----------|
| `AUTO_SCENE_IR_SAT_MAX` | 25 | mean HSV-S < N → IR candidate |
| `AUTO_SCENE_IR_CONF` | 0.10 | CONF_THRESH при IR сцене |
| `AUTO_SCENE_IR_MOT_THRESH` | 8 | NIGHT_MOT_THRESH при IR |
| `AUTO_SCENE_IR_DIFF_THRESH` | 6 | NIGHT_DIFF_THRESH при IR |

**pipeline.py** — метод `_adapt_auto_scene`:
- Классификация (приоритет):
  1. `mean Y < NIGHT_BRIGHTNESS_MAX` → `'night'`
  2. `mean HSV-S < IR_SAT_MAX` → `'ir'`
  3. иначе → `'day'`
- Saturation вычисляется из `cv2.cvtColor(crop, COLOR_BGR2HSV)[:, :, 1]`
- Grayscale-источник → `mean_sat = 0.0` → автоматически попадает в 'ir'
- Гистерезис: те же `CONFIRM_FRAMES / SAMPLE_INTERVAL` подтверждений, streak сбрасывается при совпадении кандидата с текущим состоянием
- На подтверждённом 'ir': устанавливает `cfg.CONF_THRESH`, `cfg.NIGHT_MOT_THRESH`, `cfg.NIGHT_DIFF_THRESH` из IR-специфичных полей
- На 'day': восстанавливает оригиналы (сохранены при init)

### Поведение по умолчанию

`AUTO_SCENE_DETECT = False` — функция неактивна. Активируется только при нажатии «Авто».
Manual modes (Day/Night/IR) не затронуты.

### Совместимость

Canonical operator buttons и presets не изменены. IR override применяется поверх текущего
preset только когда `AUTO_SCENE_DETECT=True`.

## Риски

- `IR_SAT_MAX = 25` — эмпирический порог. Требует ручной калибровки на реальных IR-клипах.
  RGB-клипы с десатурированными сценами (туман, серое небо) могут ложно попасть в IR.
- При тёмных IR-сценах (ночное тепловизионное изображение) приоритет 'night' может сработать
  раньше IR. Это приемлемо — обе ветки снижают conf_thresh.
- Граyscale-источники (ndim != 3) полностью перехватываются в IR — осознанное поведение.
