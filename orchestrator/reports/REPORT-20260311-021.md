## Что сделано

Добавлена локальная стабилизация bbox/center presentation для ночного режима без переписывания decision loop.

### Алгоритм (TASK-021)

**config.py** — новые поля:
| Поле | Default | Описание |
|------|---------|----------|
| `SMOOTH_BBOX_ALPHA` | 0.35 | EMA alpha для позиции (cx, cy) |
| `SMOOTH_BBOX_SIZE_ALPHA` | 0.20 | EMA alpha для размера (w, h) — более медленный |
| `SMOOTH_BBOX_HOLD_FRAMES` | 4 | кадров удержания последнего bbox при пропуске детекции |
| `DISPLAY_STATE_HOLD_FRAMES` | 3 | кадров до downgrade display tracking state |

**pipeline.py** — новые поля в `__init__`:
```python
self._smooth_bbox: Optional[list[float]] = None  # [cx, cy, w, h] float EMA state
self._smooth_bbox_missing: int = 0                # кадры без детекции подряд
```

**pipeline.py** — метод `_get_smooth_display_bbox(active)`:
- Если `active` is None → инкремент `_smooth_bbox_missing`
- Если `_smooth_bbox_missing <= HOLD_FRAMES` → возвращает последний smoothed bbox (hold)
- Если `_smooth_bbox_missing > HOLD_FRAMES` → сбрасывает state, возвращает None
- Если active есть → EMA: position alpha=0.35, size alpha=0.20
- Конвертирует float EMA state обратно в int-tuple (x1, y1, x2, y2)

**pipeline.py** — метод `_get_display_tracking_state(real_state)`:
- Удерживает display state от downgrade на `DISPLAY_STATE_HOLD_FRAMES` кадров
- SCAN→TRACK (upgrade) — немедленно
- TRACK→SCAN (downgrade) — только после N кадров подряд

**pipeline.py** — `process_frame`:
```python
display_tracking_mode = self._get_display_tracking_state(tracking_mode)
smooth_active_bbox = self._get_smooth_display_bbox(active)
result = draw_frame(..., display_tracking_mode, ..., smooth_active_bbox=smooth_active_bbox)
```
- `FrameOutput.mode` по-прежнему содержит реальный `tracking_mode` (для метрик)
- `draw_frame` получает `display_tracking_mode` (для HUD/badge)

**pipeline.py** — `draw_frame`:
- Добавлен параметр `smooth_active_bbox: Optional[tuple[int, int, int, int]] = None`
- В compact_overlay ветке: если `smooth_active_bbox` передан — рисует smoothed rect+label
  вместо вызова `_draw_target(active)`

### Поведение по умолчанию

Стабилизация применяется только к display (visual) bbox в compact_overlay режиме.
Реальные координаты `active` передаются нетронутыми в tracking-логику и метрики.
На дневных сценах визуальный эффект минимален (детекция стабильная — EMA едва заметна).

## Риски

- EMA alpha=0.35 для позиции — компромисс между плавностью и задержкой отклика.
  При очень быстрых движениях цели возможен lag 2–3 кадра. Требует ручной калибровки.
- HOLD_FRAMES=4 при 25fps ≈ 160мс удержания bbox при пропуске — приемлемо для ночи,
  но может выглядеть странно при длинных затменях.
- `DISPLAY_STATE_HOLD_FRAMES=3` уменьшает мигание badge LOCK/SCAN, но badge чуть
  задерживается при реальной потере цели.
