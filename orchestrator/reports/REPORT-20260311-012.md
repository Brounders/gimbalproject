# REPORT: TASK-20260311-012-stable-cycle-preset-preflight

## Summary
В `python_scripts/run_stable_cycle.py` добавлен pre-flight шаг валидации пресета YAML
перед запуском бенчмарка. Невалидный или несуществующий пресет прерывает цикл с exit code 2.
Если `validate_profile_presets` недоступен — предупреждение и продолжение (graceful degradation).

## Changes
| Файл | Действие |
|---|---|
| `python_scripts/run_stable_cycle.py` | Добавлена функция `_preflight_validate_preset()` + вызов в `main()` |

## Техническое решение
- Прямой `import` из `validate_profile_presets` через `sys.path.insert(0, ...)` — без subprocess
- `_HAS_VALIDATOR` guard: если import упал — предупреждение, цикл продолжается
- `_preflight_validate_preset(preset_name)` → bool:
  - Нет файла → ERROR + return False
  - Нет key mapping → WARNING + return True (не блокировать)
  - Проблемы валидации → ERROR + список issues + return False
  - OK → INFO + return True
- Вызов до проверки `rtx_model.exists()` — ошибка в пресете обнаруживается первой

```
Result:
  [preflight] ERROR: preset file not found: .../configs/nonexistent_preset.yaml
  [stable_cycle] Aborted: preset "nonexistent_preset" failed pre-flight validation.
  exit code 2
```

## Residual Risks
- `sys.path.insert(0, str(Path(__file__).parent))` добавляет `python_scripts/` в путь при каждом импорте.
  Это не идемпотентно при повторных вызовах. Для нашего use-case (один запуск) приемлемо.
- Без PYTHONPATH `build_type_rules()` упадёт с предупреждением но не ошибкой →
  пресет проверяется только на unknown keys.

## Варианты эволюции
1. Добавить `--skip-preflight` флаг для экстренного пуска без валидации.
2. Расширить `_preflight_validate_preset()` для проверки всех пресетов из `--ab-profiles`.
3. Интегрировать вывод pre-flight в итоговый JSON-отчёт stable cycle для трейсабильности.
