## Что сделано

Исправлена readiness-логика в `python_scripts/training_conveyor.py` (lines 239–240).

**Причина бага:** `ready = dataset_yaml is not None` — единственный критерий готовности.
`drone_bird_mendeley_ir_mix_v1` имел `dataset.yaml`, но `image_files=0, label_files=3` —
датасет неполный, но помечался `ready_for_training=true`.

**Фикс:** добавлена проверка `has_content = image_files > 0 or video_files > 0`.
Три ветки: нет yaml → `blocked_no_dataset_yaml`; yaml есть но нет файлов → `blocked_no_images`;
yaml + контент → `queued`.

```python
has_content = image_files > 0 or video_files > 0
if dataset_yaml is None:
    ready, status = False, "blocked_no_dataset_yaml"
elif not has_content:
    ready, status = False, "blocked_no_images"
else:
    ready, status = True, "queued"
```

## Что проверено

Inline тест на 4 случаях:
- `yaml=False, img=0, vid=0` → `ready=False, blocked_no_dataset_yaml` ✓
- `yaml=True, img=0, vid=0` → `ready=False, blocked_no_images` ✓ (целевой случай)
- `yaml=True, img=0, vid=1` → `ready=True, queued` ✓ (video-only датасет не блокируется)
- `yaml=True, img=100, vid=0` → `ready=True, queued` ✓

`python3 -m compileall -q python_scripts` → 0 ошибок.
