## Что сделано

Добавлен scene-aware lock-hardening, который автоматически ужесточает порог подтверждения lock при переходе в night или IR сцену (AUTO_SCENE_DETECT mode).

### Изменения

**`src/uav_tracker/config.py`** — два новых поля в секции auto scene detection:

```python
AUTO_SCENE_NIGHT_LOCK_CONFIRM: int = 8      # LOCK_CONFIRM_FRAMES в night/IR (default 5)
AUTO_SCENE_NIGHT_DRONE_LOCK_SCORE: float = 0.75  # DRONE_LOCK_SCORE_MIN в night/IR (default 0.62)
```

**`src/uav_tracker/pipeline.py`** — два новых поля в `__init__`:

```python
self._auto_scene_orig_lock_confirm = int(cfg.LOCK_CONFIRM_FRAMES)
self._auto_scene_orig_drone_lock_score = float(cfg.DRONE_LOCK_SCORE_MIN)
```

В `_adapt_auto_scene()` — при переходе в 'night' или 'ir' применяется override:

```python
self.cfg.LOCK_CONFIRM_FRAMES = int(getattr(self.cfg, 'AUTO_SCENE_NIGHT_LOCK_CONFIRM', 8))
self.cfg.DRONE_LOCK_SCORE_MIN = float(getattr(self.cfg, 'AUTO_SCENE_NIGHT_DRONE_LOCK_SCORE', 0.75))
```

При возврате в 'day' — восстанавливаются оригинальные значения.

### Механика fix

- **До fix**: при переходе в night/IR применялись только CONF_THRESH, NIGHT_MOT_THRESH, NIGHT_DIFF_THRESH. LOCK_CONFIRM_FRAMES оставался на 5 (5 последовательных хитов). Любая шумовая последовательность из 5 детекций могла войти в lock.
- **После fix**: в night/IR нужно 8 подряд хитов + drone_score ≥ 0.75 (был 0.62) для подтверждения lock. Это снижает вероятность входа в lock по шумовым детекциям на ночных/IR кадрах.

## Риски

- Fix активируется **только** при `AUTO_SCENE_DETECT=True` (Auto operator mode). Статичные presets (night/ir) не затронуты — оператор может вручную задать `LOCK_CONFIRM_FRAMES: 8` в YAML.
- Увеличение LOCK_CONFIRM_FRAMES с 5 до 8 увеличивает время до первого lock примерно на 3 кадра (~0.1с при 30fps). Это приемлемо.
- Targeted smoke на реальных видео не запускался в рамках этого task (TASK-031 закрывает это).
