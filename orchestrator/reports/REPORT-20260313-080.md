## Verdict

**`partial_improvement`**

### Что улучшилось
- **id_chg/min для `night_ground_large_drones`: −44%** (55.05 → 30.58)
- **Contract gap по id_chg: с 3.06× до 1.70×** — gap вдвое сократился
- **`night_ground_indicator_lights`: без регрессии** (false_lock=0.458, id_chg=0.00)

### Что не улучшилось
- **large_drones false_lock: marginal regression** (+0.021, 0.750→0.771)
  Acceptance criterion `false_lock < 0.750` не достигнут.

### Подтверждённые гипотезы
1. Night detector layer — правильный target для id_chg/min. Все 13 lock policy knobs (AP-018..022)
   дали best point 48.93/мин. Один night detector knob (`night_track_dist=65`) дал 30.58/мин.
2. `night_max_area` не оказывает измеримого эффекта на данном клипе — blob area < 200 px².
3. `night_lost_max` управляет false_lock (увеличение удерживает false locks дольше → рост false_lock).
4. `night_track_dist` и false_lock антагонистичны: снижение id_chg за счёт маргинального false_lock роста.

### Следующий шаг
- false_lock на large_drones (0.771) требует дополнительного исследования:
  возможно, fine-tuning диапазона `night_track_dist` (47–58) даст точку с id_chg < 48 и false_lock ≤ 0.750.
- Альтернативно: исследовать `NIGHT_CONFIRM` (default: 3) как дополнительный knob.
