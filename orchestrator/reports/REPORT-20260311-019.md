## Что сделано

Восстановлено базовое операторское отображение: **рамка + ID цели**, без прицела по умолчанию.

### Root cause

В `modes.py` для режимов `operator` и `embedded` было задано `RETICLE_OVERLAY_ENABLED = True`.
В `draw_frame` reticle-блок выполняется ПЕРВЫМ и возвращает кадр досрочно (`return frame`), обходя
секцию compact_overlay (bbox+ID). Это делало reticle неустранимым в операторском режиме.

### Исправление

`src/uav_tracker/modes.py`:
- `operator` mode: `RETICLE_OVERLAY_ENABLED = True` → `False`
- `embedded` mode: `RETICLE_OVERLAY_ENABLED = True` → `False`

После этого в operator mode работает compact_overlay секция:
- `_draw_target(frame, ..., compact=True)` рисует рамку с `[LOCK] ID:X`
- Строка `FPS: X.X  TRACK  ID X` в нижней части

Reticle остается включенным только в `research` mode (для разработчиков).

## Какие файлы изменены
