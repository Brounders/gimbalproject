# REPORT: TASK-20260311-003-package-import-decoupling

## Summary
Убран eager-import `pipeline` из `src/uav_tracker/__init__.py`. Заменён на `__getattr__` lazy-pattern
с кэшированием в `globals()`. Символы `TrackerPipeline`, `apply_runtime_preset`, `run_tracker`
по-прежнему доступны через `from uav_tracker import ...`, но загружаются только при первом обращении.
`Config` остаётся eager-экспортом без изменений.

## Changes

| Файл | Действие |
|---|---|
| `src/uav_tracker/__init__.py` | Убран eager `from uav_tracker.pipeline import ...`; добавлен `__getattr__` |
| `tests/__init__.py` | Создан (пустой package init) |
| `tests/test_package_import.py` | Создан — 3 regression-теста на import boundary |
| `tests/README.md` | Создан — инструкция по запуску |

## Техническое решение

```python
from uav_tracker.config import Config

__all__ = ['Config', 'TrackerPipeline', 'apply_runtime_preset', 'run_tracker']

def __getattr__(name: str):
    if name in ('TrackerPipeline', 'apply_runtime_preset', 'run_tracker'):
        from uav_tracker.pipeline import TrackerPipeline, apply_runtime_preset, run_tracker
        globals()['TrackerPipeline'] = TrackerPipeline
        globals()['apply_runtime_preset'] = apply_runtime_preset
        globals()['run_tracker'] = run_tracker
        return globals()[name]
    raise AttributeError(f"module {__name__!r} has no attribute {name!r}")
```

Первый доступ к `TrackerPipeline`/`apply_runtime_preset`/`run_tracker` кэшируется в `globals()`,
поэтому последующие обращения не вызывают повторный import.

## Residual Risks
- `from uav_tracker import TrackerPipeline` по-прежнему тянет cv2/ultralytics при первом вызове —
  это ожидаемое поведение для runtime-точек входа.
- Если pipeline.py изменит публичный API, `__getattr__` не сломается — ошибка появится только
  при фактическом обращении к символу.

## Варианты эволюции
- Добавить smoke-test на lazy-load: проверить что `'cv2'` не в `sys.modules` после
  `import uav_tracker` (требует отдельного subprocess для изоляции).
