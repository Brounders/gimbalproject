## Ключевые наблюдения из sweep

1. **night_max_area (200→500)** не оказывает измеримого эффекта на оба KPI.
   Вывод: NIGHT_MAX_AREA=200 достаточен для данного клипа; dominant blob area < 200 px².

2. **night_lost_max** — главный driver false_lock:
   - lost_max=12: large_drones false_lock=0.799 (ухудшает — false locks удерживаются дольше)
   - lost_max=8 (default): large_drones false_lock=0.760–0.771 (лучше)

3. **night_track_dist** — главный driver id_chg/min:
   - track_dist=42 (default): 55.05/min (baseline)
   - track_dist=65: 24.47–36.70/min (улучшение ~33–56%)
   - track_dist=55–60: 30.58/min

4. **Антагонизм метрик:** более широкий track_dist снижает id_chg (меньше breaks),
   но вызывает маргинальный рост false_lock (небольшое увеличение false associations).
