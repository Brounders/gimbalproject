## Что сделано

Исправлен `python_scripts/run_quality_gate.py` — добавлен `try/except` вокруг вызова
`evaluate_source()` в главном loop (lines 194–200).

**Причина бага:** `evaluate_source(...)` вызывался без обработки исключений. Если один
клип не найден, видеофайл повреждён или evaluate падает по любой причине — исключение
поднималось на уровень `main()`, loop прерывался, `rows` оставался неполным, и ни JSON,
ни CSV aggregate файлы не записывались вообще.

**Фикс:** при исключении в `evaluate_source()`:
1. Ошибка логируется: `[error] clip_name: ExceptionType`
2. В `rows` добавляется "failed" строка с `passed=False, fail_reasons="evaluate_error:ExceptionType"`
3. В `failures` добавляется запись
4. Loop продолжается следующим клипом через `continue`
5. После loop — aggregate JSON и CSV всегда записываются

```python
try:
    report = evaluate_source(cfg, source=source, ...).to_dict()
except Exception as exc:
    print(f"[error] {source_name}: {exc}")
    failed_row: dict[str, Any] = {
        "source": str(source), "scene": scene, "frames": 0,
        "avg_fps": 0.0, ...,
        "passed": False,
        "fail_reasons": f"evaluate_error:{type(exc).__name__}",
    }
    rows.append(failed_row)
    failures.append(f"{source}: evaluate_error:{type(exc).__name__}")
    continue
```

## Что проверено

`python3 -m compileall -q python_scripts` → 0 ошибок.

Логика верифицирована чтением: после try/except loop всегда доходит до блока
`means = {...}` (line 273) и `out_json.write_text(...)` / `_csv_write(...)` (lines 313–314).
Gate result `passed = len(failures) == 0` корректно отражает ошибку клипа.

## Поведение при ошибке

До фикса: `[crash] -> no output files`
После фикса: `[error] clip_name: FileNotFoundError -> aggregate JSON written with 1 failed row, gate=FAIL`

Failed row в CSV имеет `frames=0, avg_fps=0.0` — это гарантирует что aggregate mean
будет занижен, gate не пройдёт, но артефакт будет создан и проверяем.

## Риски

Если все клипы упадут — `rows` содержит только failed rows с нулями. `means` будет
`avg_fps=0.0` итд. Gate провалится с `gate_passed=false`. Это корректное поведение.
