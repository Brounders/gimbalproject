# REPORT: TASK-20260311-011-validator-auto-types-from-config

## Summary
Устранено дублирование типовых правил в `validate_profile_presets.py`. Ручной `_TYPE_RULES` (~80 записей)
заменён на авто-вывод через `dataclasses.fields(Config)`. Graceful degradation при отсутствии PYTHONPATH.
Добавлены `extract_mapping_dict()` и `build_type_rules()`. Тесты обновлены для передачи `type_rules`.

## Changes
| Файл | Действие |
|---|---|
| `python_scripts/validate_profile_presets.py` | Удалён `_TYPE_RULES`; добавлены `extract_mapping_dict()`, `_annotation_to_yaml_types()`, `build_type_rules()` |
| `tests/test_validate_profile_presets.py` | Обновлены setUp + validate_file вызовы для передачи `type_rules` |

## Техническое решение
- `extract_mapping_dict()` → `{yaml_key: CONFIG_ATTR}` через ast (расширение `extract_mapping_keys`)
- `_annotation_to_yaml_types()` → конвертирует Config field annotation в `tuple[type, ...]`:
  - `bool` → `(bool,)`, `int` → `(int,)`, `float` → `(int, float)` (YAML целое допустимо для float поля)
  - `Union[...]` → объединение типов без None, сложные типы → None (пропускаются)
- `build_type_rules()` → опционально импортирует Config, строит правила из `dataclasses.fields()`
- При import error: предупреждение + пустой dict (type checks отключены)

## Residual Risks
- `_annotation_to_yaml_types()` пропускает `Union[int, str]` (VIDEO_SOURCE) — не маппируется в YAML пресет, OK.
- Если Config добавит новое поле, оно автоматически появится в правилах при следующем запуске.

## Варианты эволюции
- Кэшировать `type_rules` между запусками в JSON-файл для offline использования без PYTHONPATH.
