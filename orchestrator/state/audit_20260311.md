# Engineering Audit — 2026-03-11

## Scope
- Audit of completed work and current repository state for GimbalProject.
- Focus: orchestration discipline, codebase maintainability, tracking stability pipeline.

## What Is Working Well
1. Orchestrator protocol is now explicit (`active_plan.md` is execution source of truth).
2. Claude implementation loop produced validated test artifacts (TASK-003/004 accepted).
3. Consistency checks are automated in CI (`check_orchestration_state.py`).
4. Core runtime separation exists (`src/uav_tracker/runtime/*`, `tracking/*`, `detectors/*`).

## Weak Points (Priority)
1. **P1 — Monolith risk in core files**
   - `app/main_gui.py` ~1737 LOC
   - `src/uav_tracker/pipeline.py` ~1137 LOC
   - High change-coupling risk and slower review cycles.
2. **P1 — Entry-point ambiguity at repository root**
   - Coexisting launch/debug scripts (`main_tracker.py`, `tracker_gui.py`, `real_tracker.py`, `train_script.py`, `benchmark.py`).
   - Increases operator and onboarding error probability.
3. **P2 — Limited CI quality depth**
   - CI validates syntax + orchestrator consistency, but no mandatory unit test stage yet.
4. **P2 — Manual training status intake**
   - `TRAIN-20260311-001` still requires manual sync of RTX result into Mac orchestration state.
5. **P3 — Large local artifact footprint**
   - Local directories (`datasets`, `runs`, `tracker_env`) dominate storage and raise accidental-operation risk.

## Global Goals (Next Phase)
1. **Stability First:** lock/ID behavior remains deterministic across day/night/IR scenarios.
2. **Operational Clarity:** one canonical launch path for operator, one for training, one for evaluation.
3. **Controlled Evolution:** decompose large modules in small reversible steps without runtime regressions.
4. **Deterministic Training Loop:** RTX -> export -> Mac intake -> benchmark -> gate decision as a fixed cycle.

## Local Goals (Current Sprint)
1. Finalize canonical entrypoint contract and runbook for operators and engineers.
2. Add fast quality smoke stage for KPI sanity before full benchmark cycle.
3. Prepare decomposition map for `main_gui.py` and `pipeline.py` (stage-0 architecture slicing).
4. Close/open-training sync by ingesting latest RTX status into orchestrator state.

## Exit Criteria For This Audit
- Weak points prioritized and documented.
- Global/local goals published as actionable plan.
- Next task batch created in `orchestrator/tasks`.
