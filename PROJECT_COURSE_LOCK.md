# Курс Проекта (Жесткая Фиксация)

## Миссия
Разработать максимально стабильную и практичную систему обнаружения и сопровождения БПЛА в дневном и ночном небе с использованием доступных вычислительных ресурсов.

## Целевая платформа
- Текущая разработка: macOS (M1), Python, PySide6, Ultralytics.
- Обучение модели: отдельный RTX-ноутбук (CUDA).
- Целевое внедрение: Raspberry Pi 5 + Hailo (AI HAT+), с сохранением логики трекинга.

## Непереговорные правила
1. Главный KPI — стабильность сопровождения цели (не количество эпох).
2. Любой цикл обучения завершается quality-gate на фиксированном наборе видео.
3. Трекинг и обучение идут раздельными контурами:
- RTX: только train/resume/артефакты.
- Mac: код, GUI, benchmark, анализ.
4. Для обучения допускается режим `drone-first`; отсутствие класса `bird` не блокирует цикл, но учитывается как риск ложных целей.
5. В процессе активного run запрещены ручные `kill` дочерних train-процессов, кроме аварии по температуре.

## Текущий стратегический курс
1. Стабилизировать training-loop на RTX (повторяемость, без дублей/срывов).
2. На каждом run получать артефакты: `best.pt`, `last.pt`, `results.csv`, log.
3. На Mac запускать benchmark/quality-gate и принимать решение `accept/reject`.
4. Параллельно улучшать pipeline трекинга (SCAN/TRACK/LOST, anti-jitter, false-lock control).
5. После стабильного качества на Mac обеспечить parity-поведение для RPi5 + Hailo.

## Что считается прогрессом
1. Снижение `active_id_changes` и `false_lock_rate`.
2. Рост/стабильность `continuity` и `active_presence`.
3. Отсутствие деградации FPS ниже операционного минимума.
4. Повторяемый run без ручной отладки инфраструктуры.

## Что считается отклонением
1. Фокус на “докрутку эпох” без quality-gate.
2. Смешивание изменений кода и обучающих экспериментов в одном цикле.
3. Нестандартизованный запуск обучения (разные python/path/wrapper без контроля).
4. Изменения, не влияющие на KPI сопровождения.

## Веха перехода к RPi/Hailo
Переход к embedded-переносу выполняется, когда:
1. Есть стабильный baseline-модельный цикл на RTX.
2. KPI трекинга на фикс-паке клипов прошли внутренний gate.
3. Контракт данных pipeline и метрик стабилен.
