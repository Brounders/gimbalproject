# Операционный Протокол Сессий (Mac + RTX)

## Роли сессий
1. RTX-сессия:
- Запуск обучения (`train/resume`).
- Контроль термального профиля.
- Подготовка артефактов после завершения run.

2. Mac-сессия:
- Разработка кода трекера и GUI.
- Benchmark и quality-gate.
- Принятие решений по следующему циклу.

## Стандарт одного цикла
1. `Start/Resume` на RTX по фиксированному шаблону.
2. Контроль run:
- прогресс epoch/batch;
- обновление `last.pt`;
- температурный контроль.
3. Финализация run:
- сбор `best.pt`, `last.pt`, `results.csv`, log.
4. Передача артефактов на Mac.
5. Benchmark и quality-gate.
6. Решение:
- `accept` (новый baseline),
- `reject` (следующий цикл с корректировками).

## Правила запуска обучения
1. Использовать только интерпретатор venv проекта.
2. Один логический train-контур на один `run_name`.
3. Любой resume — от `last.pt`.
4. Термальный режим важнее максимальной скорости.

## Минимальный отчет после каждого run
1. `status`
2. `run_name`
3. `run_dir`
4. `best_pt`
5. `last_pt`
6. `results.csv`
7. `tail_log`
8. краткий вывод по стабильности run

## Память и контекст (обязательная синхронизация)
Перед началом новой итерации перечитывать:
1. `PROJECT_COURSE_LOCK.md`
2. `PROJECT_NAVIGATOR.md`
3. `DEVELOPMENT_NEXT_STEPS.md`
4. актуальный статус последнего run (артефакты + benchmark)

## Запрещенные действия в активном run
1. Ручной `kill` дочерних train-процессов без аварийной причины.
2. Параллельный запуск второго train для того же `run_name`.
3. Изменение параметров запуска в середине цикла.
